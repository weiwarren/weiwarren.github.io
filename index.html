Modal Dialog

Modal Dialog allows applications to show a dialog of HTML content that freezes all other content while showing. Chrome and other browsers are removing its support.

Analysis

  • Whenever a page requests a modal dialog to open, any JavaScript execution in the originating page completely paused until the modal is closed.
  • An opener page can pass initialization values through a parameter that creates a window.dialogArguments property on the modal window.
  • The modal window can then return values back to its opener document by assigning a value to window.returnValue.

Remediation

The ShowModalDialog Polyfill can be used to provide support for this functionality in Chrome: View project .

This project leverages the "Dialog" HTML5 element to display the dialog and stop code execution while it is open (when window.showModalDialog is not supported).

It also complies to the window.ShowModalDialog interface. 

           
function() {
    //statements before showing a modal dialog
    var rt = window.showModalDialog( url [, arguments, options] );
    //statements after closing a modal dialog
});

XML Data Island

XML Data Islands can be classified as a Data Source Object within IE's legacy Data Binding Architecture.

Their support has been removed in IE 10 and they do not work in Chrome. 

           
<XML id="xmlDBCommon"></XML>
<XML id="xmlDBError"></XML>
<XML id="xmlDataCSHiCashNewUnBankedGet"></XML>

Analysis

  • Whenever IE finds an "xml" markup element, it creates an in-memory Data Source object that can be used for data binding and data manipulation.
  • The object's data can be initialized from an external file or service location or it can be embedded as XML inside the "xml" tag.
  • The Data Island in-memory object provides two interfaces to interact with the contained data: an ActiveX RecordSet object and a IXMLDOMDocument2 direct interface to the XML, of which none are cross browser.

Remediation

Create a custom js prototype which can act as the in-memory manager for the Data Islands. This object has to rely on two other custom prototypes for the IXMLDOMDocument2 interface and the ActiveX RecordSet object.

Create an AngularJs element directive ("xml") that can trigger the instantiation of a custom Xml Data Island manager object in the window scope.

Use a custom css style to make the "xml" tag and its contents invisible to the user interface.

XML Data Island IXMLDOMDocument2

Data Islands expose access to the contained data through the Microsoft IXMLDOMDocument2 interface.

This interface is implemented by the Microsoft.XMLDOM ActiveX component and gives XPath, XSLT and data manipulation to XML documents.

XML Data Island RecordSet

Data Islands also expose access to the contained data through the Microsoft ADODB.Recordset interface.

This interface is implemented by the ADODB.Recordset ActiveX component and gives table cursor style access to the XML document. 

           
             xmlDataIsland.recordset.[ADODB.Recordset]

Analysis

  • Used to query and modify the underlying XML contained in the Data Islands.
